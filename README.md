## Commit Convention
- 기본 구조 
```
[#Issue-Number] <Type>: <Message>
```
- 간단한 커밋 
```
[#1] feat: 기능 추가
```
- 부가적인 설명이 필요한 커밋 
```
[#1] feat: 기능 추가
- 작업1
- 작업 2
```

- `feat`: 기능
- `fix`: 버그 수정
- `env`: 환경 변수 또는 프로젝트 환경
- `ref`: 리팩토링
- `style`: 스타일(코드 형식, 세미콜론 추가: 비즈니스 로직에 변경 없음)
- `docs`: 문서 (문서 추가, 수정, 삭제)
- `test`: 테스트(테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없음)
- `chore`: 기타 변경사항 (빌드 스크립트 수정 등)
